# Framework Analysis

深度分析软件项目框架的标准流程和报告模板。

## 触发条件

- 用户请求分析某个项目或框架
- 用户需要对比两个框架的差异

---

## 分析流程

### Phase 1: 项目探索
- 列出项目目录结构
- 识别项目类型和配置文件
- 阅读 README 和文档

### Phase 2: 架构分析
- 目录结构和分层架构
- 设计模式识别：Registry, Factory, Strategy, Stage/Pipeline
- 依赖关系

### Phase 3: 代码实现分析
- 优势：架构亮点、性能优化、最佳实践
- 问题：复杂度、风险、设计缺陷
- 关键组件深入

### Phase 4: 部署分析
- 部署方式：API、服务化、容器化
- 生产特性：分布式、容错、监控
- 风险评估

### Phase 5: 对比分析（可选）
- 架构设计、并行策略、性能优化、部署方式、生态支持

## 报告模板

```markdown
# [项目名称] 框架深度分析报告

## 一、框架概述
- 核心定位、支持模型/功能

## 二、架构设计分析
- 整体架构图
- 核心设计模式
- 关键组件设计

## 三、代码实现分析
- 优势（评价等级）
- 问题与风险

## 四、工业化部署分析
- 部署方式
- 生产部署优势/风险
- 部署建议

## 五、框架对比（可选）
| 维度 | 框架A | 框架B |
|------|------|------|
| ... | ... | ... |

## 六、总结与建议
- 优势（✅）/ 不足（❌）
- 改进建议：架构、代码、运维
- 适用场景：推荐 / 谨慎使用

---
*报告生成时间: [日期]*
```

## 评价等级

| 等级 | 含义 |
|------|------|
| 优秀 | 行业领先，值得学习 |
| 良好 | 实现完善，可参考 |
| 一般 | 基本可用，有改进空间 |
| 待改进 | 存在问题，需要优化 |

---

## 常见架构模式

| 模式 | 特征 |
|------|------|
| Registry | `@MODEL_REGISTER("name")` 装饰器注册 |
| Factory | `Factory.create(type)` 工厂方法 |
| Strategy | 可替换的策略实现 |
| Stage/Pipeline | 可组合的阶段式处理 |

---

## 对比分析维度

| 类别 | 维度 |
|------|------|
| 架构 | 分层设计、模块划分、设计模式 |
| 并行 | TP/SP/PP/DP、分布式策略 |
| 性能 | 缓存、量化、算子优化 |
| 部署 | API、服务化、容器化 |
| 生态 | 模型支持、硬件支持、社区 |

---

## 输出要求

1. **有据可依**：每个结论要有代码引用支撑
2. **客观中立**：优缺点都要分析
3. **可操作性**：建议要具体可行
